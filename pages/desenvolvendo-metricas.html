Desenvolvendo Métricas

O padrão IEEE 1061 define uma metodologia de desenvolvimento para atributos de qualidade de software. O padrão define um atributo como "a propriedade mensurável física ou abstrata de uma entidade". O fator qualidade é um tipo de atributo, "um atributo de software orientado à gestão que contribui para a sua qualidade". A métrica é a função de medição, e um métrica de qualidade de software é uma "função que suas entradas são dados de software e que a saída é um único valor numérico que pode ser interpretado como o grau no qual o processo de software possui em um atributo que que afeta sua qualidade.”

O padrão 1061 define vários critérios de avaliação, que se resumem nos seguintes:

  1. Correlação: A métrica tem que ter uma correlação estatística linearmente relacionada com o fator de qualidade.
  2. Consistência: Seja F a variável do fator de qualidade e Y a saída da função de métrica, M: F→Y. M deve ser uma função monotônica. Isso é, se f1 > f2 > f3, então nós devemos obter y1 > y2 > y3;
  3. Rastreamento: Para a função de métrica, M: F→Y. Como F muda de ff1 para f2 em tempo real, M(f) pode mudar imediatamente de y2 para y2.
  4. Previsibilidade: Para funções de métrica, M: F→Y. Se soubermos o valor de Y a algum ponto, devemos ser capazes de predizer o valor de F.
  5. Poder Discriminador: Uma métrica pode ser capaz de discriminar entra componentes de alta qualidade e componentes de software de baixa qualidade. O conjunto de valores de métrica associados com a primeira deve ser significantemente maior (ou menor) que aqueles associados com a segunda.
  6. Confiabilidade: "Uma métrica deve demonstrar a correlação, rastreamento, consistência, previsibilidade, e propriedades de poder discriminador para ao menos P% da aplicação da métricas."
