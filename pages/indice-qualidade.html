ÍNDICE DE QUALIDADE ESTRUTURAL DO PROJETO (DSQI)

É uma métrica de projeto arquitetural usada para avaliar a estrutura do projeto de softwares e da eficiência de seus módulos (e.g. pacotes de software orientados a objetos). Possui a vantagem de ter o cálculo de suas métricas bem transparente e, é idealmente utilizado com algum framework de extreme programming.

Os aspectos de sistema de software usados no cálculo da métrica são os seguintes:

  1. O total de números de módulos definidos na arquitetura do programa.
  2. O número de módulos cujas funções corretas dependem de uma entrada ou produzam uma saída para ser usada em outro local.
  3. Acoplamentos eferentes (Ce): o número de classes de outros pacotes, cujas classes no pacote analisado são dependentes, é um indicador da dependência em externalidade do pacote.
  4. O número de módulos cujas funções corretas dependem de processamento prévio.
  5. O número de itens no banco de dados.
  6. O número total de itens únicos no banco de dados.
  7. O número de segmentos do banco de dados.
  8. O número de módulos com um único ponto de entrada e de saída.

Exemplo:

  1. Como esse projeto foi desenvolvido usando orientação à objeto, o numero de classes pode ser usado. (24)
  2. Uma parte das funcionalidades dos módulos dependem de outra fonte ou produzem dados para outro modulo (4 classes recebem dados de módulos que recebem informação de uma conexão web). (4)
  3. Número de módulos que requerem dados serem previamente processados por outros módulos (temos 4 classes que populam abas baseadas nessas informações dos 4 módulos mencionados anteriormente). (4)
  4. Todos os itens do banco de dados, incluindo objetos e atributos que definem os objetos. (22)
  5. Todos os elementos do banco de dados que não são duplicados ou similares a outros itens. (5)
  6. Número de segmentos do banco de dados, incluindo diferentes registros ou objetos únicos. (80)
  7. Módulos que não possuem múltiplos pontos de entrada e saída. (0)

Já que o método de implementação usado foi de orientação à objeto, marcamos a opção de método distinto.

A tela mostra os pontos que o sistema contém, para cada aspecto da estrutura do sistema, A média total é .301 baseado nas prioridade dos aspectos e nas notas que foram dadas. As vantagens dessa métrica é que é fácil de calcular e pode ajudar desenvolvedores a verem onde o design deles podem dar problema no futuro. O número por si só não é muito útil, o ideal é que ele seja comparado com o de um sistema que foi sucesso. Quanto mais próximo de 1 melhor o índice.
